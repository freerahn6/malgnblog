# 맑은이러닝 범용 LMS 제품 지식 시트 (블로그팀 SSOT) — b1~b4 필독

> 블로그 집필 시 **"맑은이러닝 LMS가 실제로 무슨 기능을 가졌나"의 단일 출처**. 기능명·화면 흐름은 여기 적힌 **관리자 매뉴얼 실제 표기**만 인용한다. 여기 없는 사양·수치·기능은 쓰지 말 것(추정·창작 금지, 불확실은 `[확인필요]`).
> 근거: `C:\job\data\매뉴얼\개인 페이지 & 공유된 페이지\범용 LMS 관리자 매뉴얼\` 관리자 매뉴얼(관리자 화면 캡처 실사). 제품 정식 명칭은 관리자 화면 상단 로고 기준 **"교육통합관리시스템"**.

## 0. 레드라인 상속 (`_facts.md`에서 그대로 가져옴 — 위반 시 발행 차단)
- ⛔ **영구 금지**: "교육효율 200%↑ / 관리시간 50%↓" — 근거 없는 마케팅 카피. 정보·실무·홍보 어디에도 금지.
- **CSAP**: 스코프 없는 "국내 유일 CSAP LMS" 금지. 반드시 ✅"국내 **최초** CSAP 인증 LMS"(역사적 사실) 또는 ✅"**상용 LMS 중 유일한** CSAP 인증 LMS"(현재 기준)로 한정.
- **미검증 수치·미동의 고객 실명/로고·경쟁사 실명 비방 금지**. 고객은 업종 유형(대학/기업/공공/협회)으로만.
- 사용 가능 팩트(요약): 누적 800여 개사 / 현재 약 375개 기관 / 동시접속 약 5,000명 / 데이터 4중 백업 / 16년(자사 표기). 상세·정확표현은 `_facts.md` 원문 확인.
- ⚠ **본 시트의 화면 캡처에 보이는 숫자는 데모 데이터**(예: Total 24건, 회원 44건, 매출 그래프 값, std001 같은 샘플 계정). **제품 실적/규모로 인용 금지.** 인용 가능한 것은 **기능명·옵션·화면 흐름**뿐.

## 1. 제품 개요
- **범용 LMS = 관리자(운영자) 페이지 + 사용자(수강생/엔드유저) 페이지** 2계층 구조.
- 관리자 화면 IA(상단 대탭): **회원정보관리 · 콘텐츠관리 · 교육상품관리 · 주문결제관리 · 홈페이지관리 · 자격관리**. 각 대탭 아래 좌측 메뉴로 세부 기능이 묶인다. 공통 UI 요소: 상단 **퀵메뉴 / 퀵메뉴추가(★)**, 화면별 **동영상가이드** 버튼, 목록 상단 **메일·SMS·쪽지·EXCEL** 일괄 액션.
- 동영상은 별도 **동영상 플랫폼**(위캔디오·콜러스·CDN(다날)·FTP)과 연동 — 상세는 b3 담당 별도 시트. 본 시트에선 "영상 저장·송출 플랫폼이 붙는다" 수준만.

## 2. 대분류별 기능 사실 (무엇을 하나 · 운영자 렌즈 · 블로그 인용)

> 아래 좌측 메뉴/옵션명은 관리자 화면 캡처에서 확인한 **실제 표기**다. 캡처로 확인하지 못한 세부 옵션은 `[확인필요]`.

### 2-1. 관리자 페이지 — 대시보드 / 기타 기능
- **대시보드(운영현황)**: 처리 대기 업무를 카드 위젯으로 집계 — **승인 필요한 회원 · Q&A 미답변 · 과정 Q&A 미답변 · 입금 확인 · 환불 처리 · 배송대기/준비 · 탈퇴회원**. 하단 **당월 매출현황**(입금대기/결제완료/결제취소/환불완료). **대시보드 설정** 버튼으로 위젯 구성.
- 운영자 렌즈: "오늘 뭘 처리해야 하나"를 한 화면에서 잡아준다 — 이 위젯이 없으면 승인·미답변·입금·환불을 메뉴마다 돌며 손으로 챙겨야 한다.
- 블로그 인용: "LMS 운영 대시보드에서 승인 대기·미답변·입금/환불 처리 건을 한눈에" (실무MOFU 도입부 소재).
- **기타 기능**(원문 확인): 상단바 우측 **[고객센터]·[매뉴얼]·[사이트]** 바로가기, **퀵메뉴 추가**(자주 쓰는 메뉴 즐겨찾기). — 운영 편의 기능이라 블로그 소재로는 부차적.

### 2-3. 홈페이지관리
- **게시판목록 → 게시판관리**: 타입 지정 게시판 생성(예: 목록형). 기본 게시판 **후기관리 · FAQ · 공지사항 · 자료실**. 코드/타입/게시판명으로 관리.
- **위시리스트관리**: 과정위시리스트 · 마이크로러닝위시리스트 · 상품위시리스트 · 도서위시리스트. 구분(온라인/집합/혼합/패키지), 이메일 수신동의여부까지 필터 → **메일/EXCEL** 내보내기.
  - 운영자 렌즈: 찜(위시)한 회원 명단+수신동의 상태를 뽑아 **재마케팅 메일** 발송 가능. 이 연결이 없으면 관심고객을 놓친다.
- **디자인관리**(원문 확인·승격): **팝업관리**(PC/모바일/PC+모바일, 다중 슬라이드) · **사이트템플릿설정**(폰트·메인색상, 서브비주얼 영역, **수강신청 페이지 타입: 기본형/2열형/커리큘럼형**, 대메뉴 템플릿 좌우형/상하형/상하축소형, 하단 푸터) · **페이지관리**(소개·이용약관·개인정보처리방침·환불규정·찾아오시는길, 카카오맵 삽입) · **메인화면관리**(모듈 15종: 탑배너·롤링배너·과정진열·과정롤링진열·배너+과정진열·마이크로러닝영상진열·마켓/도서상품진열·바배너·영상영역·일반영역·강사진열·수강후기진열·교육일정·게시판+배너진열·퀵배너) · **사이트메뉴관리**(대/하위 메뉴 트리). ※ **사이트 상단 메뉴 구조 변경은 별도 유지보수비용 발생 → 맑은소프트 문의** 사항.
- **사이트관리**: 교육일정관리(캘린더) · 사이트정보 · 태그관리 · 정기구독설정관리.
- **포인트관리**: 포인트정책설정 · 포인트이용내역 · 포인트쿠폰관리.
  - 포인트정책설정: **포인트단위명칭**(예: 원/학점/포인트)을 사이트에 맞춰 표기, 적립 트리거 — **로그인 / 회원가입 / 결제완료(결제액 % 부과) / 강의학습완료(차시당 최초 1회) / 과정수료(과정당 최초 1회)**. 포인트↔쿠폰 전환.
  - 운영자 렌즈: "학점"으로 명칭을 바꿔 마일리지 제도처럼 운용, 학습완료·수료에 자동 적립 → 재수강 동기 설계. 수기 적립이 필요 없다.

### 2-6. 회원정보관리
- **회원관리**: 회원관리 · 강사관리 · 관리자관리 · 회원소속관리 · 회원그룹관리 · 회원등급관리.
  - 회원관리 필터: **회원유형(회원/과정운영자/소속운영자/운영자/최고관리자)**, 상태(정상/중지/탈퇴/삭제), 회원소속.
  - **강사관리**: 유형·소속·강사명·메인노출·노출 토글, 강사정보(영문명·강사소속·전공·최종학력·은행명·계좌번호), **강사 정산 구분·강사 세율** 설정.
- **발송관리**: 메일발송관리 · 메일템플릿관리.
- 운영자 렌즈: **회원소속/그룹/등급**으로 기업·부서·차수 단위 회원을 구조화(=B2B 위탁교육·기업고객 분리 운영의 핵심). **권한 계층**(최고관리자~소속운영자)으로 위탁사 담당자에게 부분 권한 위임 가능. 메일템플릿으로 안내·독려 발송 자동화.
- 블로그 인용: "회원 소속·그룹·등급 + 다단계 관리자 권한 → 기업/기관별 분리 운영", "메일 템플릿 기반 수강 독려" — 기업교육/공공 타깃 MOFU·BOFU.

### 2-7. 자격관리 (자격검정)
- **자격 관리**: 자격개설및운영 · 검정시험관리 · 통합응시관리 · 자격증발급관리 · 자격취득후기관리 · 고사장관리. (자격명·상태(정상/중지))
- **자격결제관리**(원문 확인·승격): **검정시험결제내역**(원서접수 결제) · **자격증발급결제내역** · **환불내역** · **자격증발송내역**. 주문상태 5종(결제완료/입금대기/결제취소/부분환불/전액환불), 무통장 입금확인, **자격증 실물 배송**(송장 일괄등록·배송상태 7단계). ※ PG 결제취소는 PG사 상점관리자에서 직접 처리(시스템은 기록·확인용).
- **자격 검정 기타 세팅**(원문 확인·승격): 홈페이지 메인화면에 **자격 검정 모듈**(자격 검정 퀵메뉴 · 진행중인 원서접수 자동 노출 · 자격검정배너) 설정.
- **자격증 사용자 페이지**: 수강생(응시자) 프런트 여정.
- 운영자 렌즈: **자격 개설→검정시험→응시(통합응시)→자격증 발급→(실물 발송)→고사장 관리**까지 자격검정 운영 일습. 협회·자격기관이 별도 시스템 없이 원서접수 결제·자격증 발급·배송까지 굴릴 수 있다.
- 블로그 인용: "자격 개설·검정시험·응시접수·자격증 발급·고사장 관리 통합" — 자격검정/인증(cluster C) 실무·신뢰 글. `_facts.md`의 "자격검정 시스템" 제품군과 연결.

### 2-8. 소개 / 엔드유저(수강생 여정)
