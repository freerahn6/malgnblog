### 1.3 2단계 서브카테고리 (선택 노출 / URL에는 미반영 — 아래 주의)

> **결정:** URL은 `/{상위}/{slug}/` **단일 뎁스**로 고정한다. 서브카테고리는 **태그/필터 및 필러-클러스터 논리 구조**로만 사용하고 URL 경로에는 넣지 않는다.
> 이유: (1) 글이 성장하며 카테고리 재분류가 잦은데 URL에 2뎁스를 박으면 301 리다이렉트 부채가 커진다. (2) 얕은 URL이 크롤 효율·CTR에 유리. 2단계는 **논리적 IA(사이트맵·목록 필터·내부링크 허브)** 로만 존재.

| 상위 | 서브(논리/태그) | 대표 필러(pillar) 페이지 |
|---|---|---|
| A LMS | 구축가이드 / 선택·비교 / 운영·관리 / 클라우드·인프라 | "LMS 완벽 가이드"(허브) |
| B HRD | 법정의무교육 / 환급과정 / 사내교육 설계 / 교육성과·통계 | "기업교육 담당자 가이드" |
| C 자격검정 | 시험(CBT) / 인증서·발급 / 협회·기관 운영 | "자격검정 시스템 구축 가이드" |
| D 동영상 | OTT·플랫폼구축 / 스트리밍·CDN / 웨비나·라이브 / 영상수익화 | "기업 영상 플랫폼 구축 가이드" |
| E 에듀테크 | 교육AI 활용 / LXP·개인화 / 트렌드·리포트 | "교육 AI 트렌드 허브" |
| F 소식 | 뉴스·수상 / 고객사례 / 제품업데이트 | (허브 없음, 시계열) |

각 상위 카테고리는 **1개 필러(허브) + N개 클러스터(개별)** 구조. 필러는 `/lms/lms-complete-guide/`처럼 상위 slug 아래 대표 글로 존재하고, 클러스터 글들이 필러로 내부링크를 몰아준다(토픽 오소리티).

## 2. URL 규칙

### 2.1 기본 패턴 (확정)
```
홈            /
카테고리 목록  /{category}/                예) /lms/
글 상세       /{category}/{slug}/          예) /lms/how-to-choose-lms/
페이지네이션   /{category}/page/2/          예) /hrd/page/2/
태그 목록     /tag/{tag-slug}/             예) /tag/microlearning/
태그 페이지네이션 /tag/{tag-slug}/page/2/
검색(선택)    /search/  (클라이언트 사이드, noindex)
정적 페이지    /about/  /contact/  /privacy/
사이트맵       /sitemap.xml   /sitemap-posts.xml  /sitemap-categories.xml
RSS(선택)     /rss.xml
```

- `{category}` ∈ `lms | hrd | certification | video | edutech | news` (1.2의 slug 고정 집합)
- 글은 **카테고리 소속 1개**만 URL에 반영(주 카테고리). 다중 주제는 태그로 표현.

### 2.2 slug 규칙 (확정)
- **언어: 영문 소문자만.** 한글 slug 금지(퍼센트 인코딩으로 URL 오염·공유 시 깨짐·GSC 가독성 저하 방지).
- 형식: `[a-z0-9-]` 만 허용. 공백→하이픈, 하이픈 연속 금지, 앞뒤 하이픈 금지.
- 길이: **3~6 단어, 60자 이내** 권장. 핵심 영문 키워드 포함.
- 불용어(a, the, of, to…) 최소화하되 의미 왜곡 시 유지.
- **한글 제목 → 영문 slug 매핑표를 콘텐츠 메타(front matter)에서 필수 관리.** 자동 음역(romanize) 금지 — 검색 의도 영문 키워드로 수동 지정.
  - 예) "기업교육 담당자를 위한 LMS 선택 가이드" → `lms` / `lms-guide-for-hrd`
  - 예) "고용보험 환급과정, LMS로 운영하기" → `hrd` / `employment-insurance-training-lms`
  - 예) "자격검정 시스템 구축, 등록부터 발급까지" → `certification` / `certification-system-build-guide`
- **slug 불변 원칙:** 발행 후 slug 변경 금지. 부득이 변경 시 구 URL→신 URL **301 매핑표**(`/redirects.json` 또는 서버 규칙)에 반드시 등재.

### 2.3 트레일링 슬래시 정책 (확정)
- **디렉터리 스타일 + 트레일링 슬래시 유지**(`/lms/how-to-choose-lms/`).
- 각 글은 `.../{slug}/index.html`로 빌드되어 슬래시 URL과 자연 정합.
- **슬래시 없는 요청은 301로 슬래시 버전으로 리다이렉트**(중복 URL 방지). 호스팅(예: Cloudflare Pages/Netlify)에서 강제.
- canonical은 항상 트레일링 슬래시 포함 절대경로로 출력.

### 2.4 페이지네이션·태그 SEO
- 목록 1페이지는 `/{category}/` (정본), 2페이지부터 `/{category}/page/N/`.
- 각 페이지 canonical은 **자기 자신**(page/2는 page/2가 canonical). `rel=prev/next`는 구글이 미사용하므로 생략, 대신 각 페이지에서 개별 글로의 링크로 크롤 유도.
- 태그 페이지: 얇은 콘텐츠 위험 → **글 3개 미만 태그는 생성 금지**. 태그 목록은 index,follow 유지하되 소개문 1~2문장 삽입(얇은 페이지 방지).

## 3. 메타·구조화데이터 설계

### 3.1 `<title>` / meta description 템플릿

**title (권장 30~40자, 최대 60자/약 600px)**
```
{핵심키워드 + 실무 베네핏} | 맑은소프트 블로그
```
- 글 상세 예: `LMS 선택 가이드 7단계 — 기업교육 담당자 체크리스트 | 맑은소프트 블로그`
- 카테고리 예: `LMS·이러닝 구축/운영 | 맑은소프트 블로그`
- 홈: `맑은소프트 블로그 — LMS·기업교육·에듀테크 실무 인사이트`
- 규칙: 브랜드 접미사(` | 맑은소프트 블로그`) 고정, 핵심키워드를 앞 30자 안에 배치.

**meta description (권장 70~80자, 최대 155자/약 920px)**
```
{글의 핵심 요약 + 검색의도 충족 문구 + 행동유도}. 맑은소프트가 정리했습니다.
```
- 예: `LMS를 처음 도입하는 기업교육 담당자를 위한 7단계 선택 가이드. 기능·비용·운영·확장성 체크리스트와 실무 팁을 정리했습니다.`
- 규칙: 핵심키워드 1회 자연 포함, 155자 초과 금지(잘림), 목록/글마다 유니크(중복 금지).

**front matter 필수 필드 (마크다운 상단 — 작가/빌드 공용 계약)**
```yaml
---
title: "LMS 선택 가이드 7단계 — 기업교육 담당자 체크리스트"   # <title>용(브랜드 접미사는 빌드가 부착)
description: "LMS를 처음 도입하는 기업교육 담당자를 위한 7단계 선택 가이드..."  # 70~80자
category: "lms"            # 고정 집합
slug: "lms-guide-for-hrd"  # 영문, 불변
tags: ["lms", "hrd", "선택가이드"]
date: "2026-07-15"
updated: "2026-07-15"
author: "맑은소프트"
funnel: "TOFU"            # TOFU|MOFU|BOFU (마케팅 배합 추적)
cluster: "A-선택비교"      # 마케팅 클러스터 매핑 [감마 조율]
thumbnail: "/assets/img/lms/lms-guide-for-hrd/cover.webp"
faq:                       # 있으면 FAQPage 생성
  - q: "LMS 도입 비용은 얼마인가요?"
    a: "규모와 방식(구축/임대)에 따라 다르며..."
---
```

### 3.2 `<head>` 공통 블록 (빌드 템플릿)
```html
<meta charset="utf-8">
<meta name="viewport" content="width=device-width, initial-scale=1">
<title>{{title}} | 맑은소프트 블로그</title>
<meta name="description" content="{{description}}">
<link rel="canonical" href="https://blog.malgnsoft.com/{{category}}/{{slug}}/">
<meta name="robots" content="index,follow,max-image-preview:large">

<!-- Open Graph -->
<meta property="og:type" content="article">
<meta property="og:site_name" content="맑은소프트 블로그">
<meta property="og:title" content="{{title}}">
<meta property="og:description" content="{{description}}">
<meta property="og:url" content="https://blog.malgnsoft.com/{{category}}/{{slug}}/">
<meta property="og:image" content="https://blog.malgnsoft.com{{thumbnail}}">
<meta property="og:locale" content="ko_KR">

<!-- Twitter -->
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:title" content="{{title}}">
<meta name="twitter:description" content="{{description}}">
<meta name="twitter:image" content="https://blog.malgnsoft.com{{thumbnail}}">
```
> 주의: og:image/twitter:image는 크롤러가 상대경로를 해석 못 하므로 **절대경로 강제**. 본문 이미지·내부링크만 상대경로. 도메인 미연결 로컬 프리뷰 단계에서는 빌드 env `SITE_URL`을 `http://localhost:4321` 등으로 주입, 배포 시 `https://blog.malgnsoft.com`으로 스위치.

```html
  "description": "국내 LMS 솔루션 기업. 범용 LMS(맑은이러닝), 위캔디오(영상 클라우드), 자격검정 시스템 제공."
}
</script>
```
> `sameAs` URL은 홍보팀 확정 SNS로 교체 `[감마 조율]`.

### 3.5 robots.txt (배포용)
```
User-agent: *
Allow: /
Disallow: /search/
Disallow: /*?*        # 파라미터 URL 크롤 차단(정렬/필터 노이즈)
Sitemap: https://blog.malgnsoft.com/sitemap.xml
```
> 도메인 미연결 프리뷰 단계에서는 `Disallow: /`로 전체 차단하거나 배포 자체를 하지 않아 색인 사고 방지. 정식 도메인 연결 시점에만 위 버전 적용.

### 3.6 sitemap.xml 방침
- 빌드가 자동 생성. 글/카테고리/정적페이지 포함, `noindex`·검색·태그 얇은 페이지 제외.
- 인덱스 분할: `/sitemap.xml`(인덱스) → `/sitemap-posts.xml`, `/sitemap-categories.xml`.
- 각 URL에 `<lastmod>`(front matter `updated`) 출력. `changefreq`/`priority`는 구글 무시하므로 생략 가능.

### 4.3 "글 1편 → 실제 페이지" 흐름
1. 작가가 `src/content/posts/{slug}.md`에 front matter(3.1) + 본문 작성.
2. `npm run build` → `dist/{category}/{slug}/index.html` 정적 생성(메타·JSON-LD·브레드크럼 자동 주입).
3. `dist/`를 브라우저로 열거나 로컬 서버(`npm run preview`)로 미리보기. **도메인 없이 완결.**
4. 배포 시 `dist/`를 호스팅에 업로드 → 이후 도메인만 연결.

### 4.4 홈/목록/카테고리 페이지 구성
- **홈(`/`)**: 상단 히어로 + 6 카테고리 진입 카드 + 최신글 12개(카드) + 필러(허브) 링크 섹션. 홈 canonical = `https://blog.malgnsoft.com/`.
- **카테고리(`/{category}/`)**: H1=카테고리명, 소개문 2~3문장(고유 텍스트로 얇은 페이지 방지), 글 카드 목록(10개/페이지)+페이지네이션.
- **글 상세**: 브레드크럼 → H1 → 메타(날짜/카테고리) → 본문 → FAQ(있으면) → 관련글 3개(같은 카테고리 내부링크) → CTA(제품 문의).

- **내부링크: 루트 상대경로** `<a href="/hrd/employment-insurance-training-lms/">`. 도메인 이동에도 불변.

## 5. 표·이미지 SEO 규칙

### 5.1 데이터 표 (이미지 금지 — 브리프 필수)
- 모든 비교/수치/체크리스트는 **HTML `<table>`** 로. 표를 캡처 이미지로 넣는 것 금지.
- 접근성/SEO용 `<caption>` 필수, `<thead>`/`<th scope>` 구조화.
- **반응형: 가로스크롤 래퍼**로 감싼다(모바일 깨짐 방지).
```html
<figure class="table-wrap" role="region" aria-label="LMS 구축 방식 비교" tabindex="0">
  <table>
    <caption>표1. 자체 구축 vs 클라우드 임대 비교</caption>
    <thead><tr><th scope="col">구분</th><th scope="col">자체 구축</th><th scope="col">클라우드 임대</th></tr></thead>
    <tbody>
      <tr><th scope="row">초기비용</th><td>높음</td><td>낮음</td></tr>
    </tbody>
  </table>
</figure>
```
```css
.table-wrap{overflow-x:auto;-webkit-overflow-scrolling:touch;}
.table-wrap table{min-width:640px;border-collapse:collapse;width:100%;}
```

### 5.2 이미지: figure/figcaption/alt
```html
<figure>
  <img src="/assets/img/lms/lms-guide-for-hrd/dashboard.webp"
       alt="맑은이러닝 관리자 대시보드에서 수강현황과 진도율을 확인하는 화면"
       width="1200" height="675" loading="lazy" decoding="async">
  <figcaption>맑은이러닝 관리자 대시보드 — 실시간 수강현황</figcaption>
</figure>
```
- **alt 규칙:** 내용 서술형(무엇을 보여주는 화면인지) + 자연스러운 키워드. "이미지", "사진", "그림" 같은 잉여어 금지. 장식 이미지는 `alt=""`.
- **width/height 명시**(CLS 방지). `loading="lazy"`(첫 화면 히어로/썸네일 제외), `decoding="async"`.
- **포맷: WebP 기본**(원본 PNG/JPG는 소스만 보관). 필요 시 `<picture>`로 대체.
- **파일명 규칙:** `{slug}-{설명}.webp`, 영문 소문자+하이픈. 예 `lms-guide-for-hrd-dashboard.webp`. 한글/공백/대문자 금지.
- **경로 규칙:** `/assets/img/{category}/{slug}/{파일명}`. 글 단위 폴더로 격리(관리·삭제 용이).
- 커버(og:image): 1200×630 권장, `{slug}/cover.webp`.

### 5.3 반응형 이미지(선택, 대형 이미지)

```html
<img src="/assets/img/.../hero-800.webp"
```

## 8. 미해결 질문 / 감마 조율 필요

1. **카테고리↔마케팅 클러스터 1:1 매핑 확정.** 본 문서 A~F slug(lms/hrd/certification/video/edutech/news)와 마케팅 `01_마케팅_SEO전략.md` 클러스터 명칭·비중이 일치하는지 교차검증 필요. front matter의 `cluster` 값 체계 합의.
2. **발행 주체·빈도** — 개발자 Git 발행 vs 비개발자 GUI. 후자면 헤드리스 CMS(Decap/Sveltia) 도입 여부 결정(4.1).
3. **호스팅 확정** — Cloudflare Pages/Netlify/Vercel vs 사내 웹서버. 트레일링 슬래시·301 규칙 적용 방식이 호스팅별로 상이.
4. **Organization `sameAs` / 본사 URL** — 홍보팀 확정 SNS·공식 URL 필요(3.4).
5. **2단계 서브카테고리를 태그로만 둘지, 향후 URL 승격할지** — 콘텐츠 200편 이상 성장 시 재검토 트리거 설정.
6. **로고/브랜드 자산 경로** `/assets/img/brand/logo.png` 실제 파일 수급(홍보팀).
