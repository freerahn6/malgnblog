# 블로그 설치·자동 갱신 요청 (서버 담당자용)

`blog.malgnsoft.com` 블로그를 서버에 올리고, 이후 내용이 바뀌면 자동으로 갱신되게 하는 작업입니다.
**최초 1회 설정 이후에는 담당자가 손댈 일이 없습니다.**

**대상 환경** — Rocky Linux 9.5 / Apache 2.4.62 / Resin 4.0.67
**레이아웃** — git 루트 `/home/blog` · 웹루트 `/home/blog/public_html` · 데이터 `/home/blog/data`

> ### 📌 사이트 파일을 따로 전달받지 않으셔도 됩니다
> 아래 **3-2단계가 최초 업로드를 겸합니다.** git이 사이트 전체(HTML 약 33개 페이지, 10MB 내외)를
> `/home/blog/public_html` 로 내려놓습니다. 이후 갱신도 같은 경로라 파일을 주고받는 과정이 없습니다.
>
> 전달받으실 것은 **이 문서와 `apache-blog.conf`, `resin-blog.xml` 세 개뿐**입니다.

---

## 1. 요약 — 서버에서 무슨 일이 벌어지나

2분마다 `git`으로 GitHub에서 완성된 정적 HTML을 내려받아 웹루트를 갱신합니다.

```
서버 cron (2분마다)
   └─ git fetch + reset ──아웃바운드 HTTPS──▶ github.com (공개 저장소, deploy 브랜치)
                          └─▶ /home/blog/public_html 갱신 (= 웹루트)

브라우저 ─┬─ 문서 요청                ──▶ Apache가 정적 파일 직접 서빙
          └─ /api/track, /api/stats  ──▶ Resin (조회수 수집)
```

**보안 관점 요약**

| 항목 | 내용 |
|---|---|
| 인바운드 포트 개방 | **80 / 443 뿐** (Resin 포트는 외부에 열지 않음 — Apache가 localhost로만 연결) |
| 서버에 저장하는 자격증명·키·토큰 | **없음** (공개 저장소라 인증 불필요) |
| 외부에 제공하는 계정 | **없음** |
| 통신 방향 | 서버 → `github.com:443` 아웃바운드 + Apache → `127.0.0.1` Resin |
| 받아오는 내용 | 정적 HTML·XML·TXT **+ JSP 2개** (아래 참조) |
| 설치하는 런타임·패키지 | **없음** (Python·Node·DB 불필요. 기존 Apache·Resin·git만 사용) |

> **JSP 2개가 실행됩니다 — 무엇인지 정확히 밝혀둡니다.**
> `public_html/WEB-INF/jsp/` 의 `track.jsp`·`stats.jsp` 둘뿐이며, 조회수를 세어
> `/home/blog/data/stats.tsv` 텍스트 파일에 기록하는 것이 전부입니다.
> DB 접속·외부 통신·프로세스 실행·파일 삭제를 하지 않습니다. 소스는 저장소에서 그대로 열람하실 수 있고,
> `WEB-INF` 아래에 있어 URL로 직접 호출되지 않습니다(서블릿 매핑으로만 접근).

**저장소:** `https://github.com/freerahn6/malgnblog` (공개) — 브랜치 `deploy`

## 2. 사전 확인

```bash
git --version                                  # 없으면: sudo dnf install -y git
curl -sI https://github.com | head -1          # HTTP/2 200 이면 아웃바운드 정상
httpd -v && httpd -M | grep -E 'proxy_http|deflate|headers|rewrite|ssl'
id blog                                        # 계정과 홈(/home/blog) 확인
```

`proxy_http`·`headers`·`rewrite`·`deflate` 가 안 보이면 `/etc/httpd/conf.modules.d/` 의 로드 여부를,
`ssl` 이 없으면 `sudo dnf install -y mod_ssl` 을 확인해주세요(나머지는 Rocky 9 기본 포함입니다).

Resin의 HTTP 포트도 확인해둡니다(기본 8080). `resin.xml` 의 `<http port="..."/>` 값입니다.

## 3. 설치 (최초 1회)

### 3-1. 디렉터리와 권한

```bash
sudo mkdir -p /home/blog/data
sudo chown -R blog:blog /home/blog

# Apache가 /home/blog 를 통과해 public_html 을 읽을 수 있어야 합니다.
sudo chmod 711 /home/blog          # 통과만 허용(디렉터리 목록 조회는 불가)
sudo chmod 755 /home/blog/data
```

> `/home/blog/data` 는 **git이 건드리지 않는 영역**입니다(조회수 집계·Resin 작업파일 보관).
> 배포로 갱신되는 것은 `public_html` 뿐입니다.

### 3-2. 사이트 내려받기 ← **이 단계가 사이트 최초 업로드입니다**

> ⚠️ `/home/blog/public_html` 에 기존 파일이 있다면 먼저 백업해주세요. 아래 명령이 덮어씁니다.

```bash
sudo -u blog git -C /home/blog init -q
sudo -u blog git -C /home/blog remote add origin https://github.com/freerahn6/malgnblog.git
sudo -u blog git -C /home/blog fetch -q --depth 1 origin deploy
sudo -u blog git -C /home/blog checkout -q -f -B deploy origin/deploy

ls /home/blog/public_html     # index.html, sitemap.xml, lms/, hrd/, WEB-INF/ 가 보이면 성공
```

`deploy` 브랜치의 트리 모양이 `/home/blog` 와 같아서, 앞으로 갱신은 `git reset` 한 번이면 끝납니다.
`.git` 은 웹루트(`public_html`)의 **바깥**에 있으므로 외부에 노출되지 않습니다.

### 3-3. SELinux (Rocky 기본값이 Enforcing이라 반드시 필요합니다)

```bash
sudo dnf install -y policycoreutils-python-utils     # semanage 가 없다면

sudo setsebool -P httpd_enable_homedirs 1            # /home 아래 문서 서빙 허용
sudo setsebool -P httpd_can_network_connect 1        # Apache → Resin 프록시 연결 허용

sudo semanage fcontext -a -t httpd_sys_content_t "/home/blog/public_html(/.*)?"
sudo restorecon -Rv /home/blog/public_html
```

이후 git으로 새로 생기는 파일은 상위 디렉터리의 컨텍스트를 물려받습니다(추가 조치 불필요).

### 3-4. Apache

동봉된 `apache-blog.conf` 를 복사하고 검사합니다.

```bash
sudo cp apache-blog.conf /etc/httpd/conf.d/blog.malgnsoft.com.conf
sudo apachectl configtest        # Syntax OK 확인 (reload 전에 반드시)
sudo systemctl reload httpd
sudo firewall-cmd --permanent --add-service=http --add-service=https && sudo firewall-cmd --reload

sudo httpd -S                    # ← 아래 설명 참고. 그냥 넘어가지 마십시오
```

> **`httpd -S` 를 꼭 확인해주세요.** 이 파일은 conf.d에서 알파벳상 앞쪽(`blog...`)이라
> 기존 `ssl.conf` 의 `_default_:443` 보다 먼저 로드될 수 있습니다. 그러면 **이 사이트가 해당 IP의
> 기본 :443 vhost가 되어**, ServerName이 맞지 않는 다른 사이트의 HTTPS 요청까지 블로그로 넘어옵니다.
> 출력에서 `*:443` 의 default server가 blog가 아닌지 확인하고, 그렇다면 파일명 앞에 `zz-` 를 붙여주세요.

파일 안에서 확인하실 곳은 두 군데입니다.

- `ProxyPass /api/ http://127.0.0.1:8080/api/` — **Resin의 실제 HTTP 포트로 맞춰주세요.**
  이미 mod_caucho로 Apache↔Resin을 붙여 쓰고 계시면, 파일 주석의 *방식 B* 를 대신 쓰시면 됩니다.
- SSL 인증서 경로 — 아직 발급 전이라면 `<VirtualHost *:443>` 블록을 통째로 지운 채 80만 띄우고
  `sudo dnf install -y certbot python3-certbot-apache && sudo certbot --apache -d blog.malgnsoft.com`
  을 돌린 뒤 이 파일로 되돌리는 편이 수월합니다.

### 3-5. Resin

동봉된 `resin-blog.xml` 의 `<host>` 블록을 운영 중인 `resin.xml` 의 `<cluster id="app">` 안,
다른 `<host>` 들과 나란히 넣어주세요. 웹앱 루트는 `/home/blog/public_html` 이고
설정 실체(`WEB-INF/web.xml`)는 배포 때 함께 갱신됩니다.

```bash
sudo mkdir -p /home/blog/data/resin-tmp /home/blog/data/resin-work
sudo chown -R blog:blog /home/blog/data
sudo systemctl restart resin        # 또는 운영 중인 기동 스크립트
```

> **Resin 실행 계정**이 `blog` 가 아니라면, 그 계정이 `/home/blog/public_html` 을 읽고
> `/home/blog/data` 에 쓸 수 있어야 합니다.
> 예: `sudo chown -R blog:resin /home/blog/data && sudo chmod 775 /home/blog/data`

기존 `resin.xml` 에서 **두 가지를 확인**해주세요.

1. **HTTP 리스너 바인딩** — 이 웹앱의 루트가 곧 DocumentRoot라, Resin의 포트(예: 8080)가 외부에
   열려 있으면 `http://<서버IP>:8080/` 으로 사이트가 통째로 다시 나갑니다(HTTPS 강제·캐시 정책 우회).
   `<http address="127.0.0.1" .../>` 로 묶거나 방화벽에서 외부에 닫아주세요.
2. **`<javac>` source 레벨** — 1.5 미만이면 조회수 JSP가 컴파일되지 않습니다(제네릭 사용).
   Resin 4 기본값은 문제없고, 옛 설정을 물려받은 서버만 해당됩니다.

### 3-6. 자동 갱신 등록

```bash
sudo tee /etc/cron.d/malgnblog >/dev/null <<'EOF'
# 맑은소프트 블로그 자동 갱신 — 2분마다 변경분 확인 후 웹루트 반영
SHELL=/bin/bash
PATH=/usr/local/bin:/usr/bin:/bin
*/2 * * * * blog cd /home/blog && git fetch -q origin deploy && git reset -q --hard origin/deploy
EOF
sudo chmod 644 /etc/cron.d/malgnblog
```

변경이 없으면 `git fetch` 가 수 KB만 주고받고 끝납니다(부하 무시할 수준).
`git reset --hard` 는 **`public_html` 안의 추적 파일만** 갈아끼웁니다 — `data/` 와 `.git` 은 그대로입니다.

> ⚠️ **`git clean` 은 절대 넣지 마십시오.** `-fdx` 를 붙이면 추적되지 않는
> `/home/blog/data`(누적 조회수·Resin 작업파일)가 통째로 사라집니다. `reset --hard` 만으로 충분합니다.

## 4. 동작 확인

```bash
# 정적 페이지
curl -sI https://blog.malgnsoft.com/ | head -1                      # 200
curl -s  https://blog.malgnsoft.com/sitemap.xml | head -2

# 조회수 수집(204가 나와야 정상)
curl -s -o /dev/null -w '%{http_code}\n' -X POST \
     'https://blog.malgnsoft.com/api/track?p=/test/'

# 집계 파일에 쌓였는지
sudo cat /home/blog/data/stats.tsv

# WEB-INF가 막혀 있는지 (403 또는 404여야 정상. 200이면 즉시 알려주세요)
curl -s -o /dev/null -w '%{http_code}\n' https://blog.malgnsoft.com/WEB-INF/web.xml

# 현재 반영된 버전
sudo -u blog git -C /home/blog log -1 --format='%h %ci %s'

# 수동으로 즉시 갱신
sudo -u blog bash -c 'cd /home/blog && git fetch -q origin deploy && git reset -q --hard origin/deploy'
```

관리자 통계는 `https://blog.malgnsoft.com/admin/` 에서 확인합니다.

> **설치 시 비밀번호를 반드시 바꿔주세요.** 기본값(`gamma`)은 공개 저장소의 `web.xml` 에 적혀 있어
> 사실상 공개된 값입니다. 서버에서 아래처럼 파일을 만들면 그 값이 우선 적용됩니다(배포 없이 변경 가능).
>
> ```bash
> sudo -u blog tee /home/blog/data/admin.pw <<< '바꿀비밀번호'
> sudo chmod 600 /home/blog/data/admin.pw          # 소유자는 Resin 실행 계정
> ```
>
> 비밀번호가 `?pw=` 쿼리스트링으로 오가므로, 동봉된 Apache 설정은 이 사이트의 접근 로그에
> 쿼리스트링을 남기지 않도록(`blog_noquery` 포맷) 되어 있습니다. `combined` 로 되돌리지 마세요.

**잘 안 될 때**

| 증상 | 확인할 것 |
|---|---|
| 페이지가 403 | `chmod 711 /home/blog` · SELinux 3-3단계 · `httpd -S` 로 vhost 인식 여부 |
| `/api/track` 이 502·503 | Resin 기동 여부 · `ProxyPass` 포트 · `httpd_can_network_connect` |
| `/api/track` 은 204인데 `stats.tsv` 가 없음 | Resin 실행 계정의 `/home/blog/data` 쓰기 권한 |
| 갱신이 안 됨 | `journalctl -t CROND \| tail` · `sudo -u blog git -C /home/blog fetch` 수동 실행 |

## 5. 중단·제거

```bash
sudo rm /etc/cron.d/malgnblog            # 자동 갱신만 중단(사이트는 그대로 유지)
sudo rm /etc/httpd/conf.d/blog.malgnsoft.com.conf && sudo systemctl reload httpd
# resin.xml 의 <host id="blog.malgnsoft.com"> 블록 제거 후 resin 재기동
```

`/home/blog/public_html` 의 파일을 지우지 않는 한 사이트 자체는 남습니다.
